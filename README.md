Произвести процедуру отладки программного обеспечения встроенными средствами среды программирования Microsoft Visual Studio.

## Используемые способы отладки Visual Studio

В ходе выполнения практической работы были применены следующие методы и инструменты отладки:

- **Точки останова (Breakpoints)** — установка, временные и условные точки останова.
- **Пошаговое выполнение (Step Into, Step Over, Step Out)**.
- **Окна наблюдения (Watch, Autos, Locals)** — отслеживание значений переменных.
- **Окно стека вызовов (Call Stack)** — анализ последовательности вызовов методов.
- **Подсказки по данным (DataTips)** — быстрый просмотр значений при наведении курсора.
- **Окно интерпретации (Immediate Window)** — выполнение выражений во время отладки.
- **Точки трассировки (Tracepoints)** — вывод данных в окно вывода без остановки выполнения.
- **Управление выполнением (Continue, Pause, Restart, Stop)**.

## 1. Отладка консольного приложения «Фибоначчи»

**Источник:**  
[Microsoft Learn — Use the Visual Studio debugger](https://learn.microsoft.com/ru-ru/training/modules/dotnet-debug-visual-studio/4-use-visual-studio-debugger)

### Что было сделано
- Создано консольное приложение для вычисления чисел Фибоначчи.
- Установлены точки останова на цикле и рекурсивном вызове.
- Выполнено пошаговое прохождение кода с анализом изменения переменных.
- Добавлены условные точки останова для остановки при определённом значении индекса.
- Использовано окно `Autos` и `Locals` для контроля переменных `n`, `prev`, `curr`.

### Результат
Обнаружено и исправлено логическое некорректное начальное присваивание при вычислении последовательности. Программа выдаёт корректные числа Фибоначчи.

## 2. Отладка консольного приложения «Галактики»

**Источник:**  
[Microsoft Learn — Debugging for absolute beginners](https://learn.microsoft.com/ru-ru/visualstudio/debugger/debugging-absolute-beginners?view=vs-2022&source=recommendations&tabs=csharp)

### Что было сделано
- Запуск приложения под отладчиком (F5).
- Установка точки останова в методе вывода информации о галактиках.
- Использование `Step Over` для пропуска детализации вспомогательных методов.
- Применение окна `Watch` для наблюдения за списком галактик и свойством `LightYearsFromEarth`.
- Анализ стека вызовов при возникновении исключения (некорректное преобразование данных).

### Результат
Выявлена ошибка «Index out of range» при обработке массива названий галактик. Ошибка исправлена добавлением проверки границ массива. Данные выводятся корректно.

## 3. Отладка консольного приложения «Буквы»

**Источник:**  
[Microsoft Learn — Tutorial: Debug C# console app in Visual Studio](https://learn.microsoft.com/ru-ru/visualstudio/get-started/csharp/tutorial-debugger?toc=%2Fvisualstudio%2Fdebugger%2Ftoc.json&view=vs-2022#create-a-project)

### Что было сделано
- Создано приложение для подсчёта количества гласных букв в строке.
- Установлена точка останова в цикле перебора символов.
- Добавлена точка трассировки для вывода текущего символа в окно `Output`.
- Использовано немедленное окно для проверки альтернативных строк без перезапуска.
- Модификация значения переменной прямо во время отладки для проверки работы алгоритма.

### Результат
Обнаружено, что программа не учитывала заглавные гласные буквы. Исправлено приведением символов к нижнему регистру. Подсчёт выполняется верно.
